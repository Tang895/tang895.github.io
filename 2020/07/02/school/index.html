第四天-爬取数据并拆解

第一次遇到反爬机制—-反调试，不过我遇到的还算简单，通过调试器禁止断点解决，这个问题使我研究了一些反调试的内容知识

JS反调试（反debug）
　　前面我们都会用到chrome中的F12去查看网页的加载过程，当这种方法用多了网站就加了反调试的策略，只要我们打开F12，就会暂停在一个‘debugger’的代码行中，无论怎么样都跳不出去，不管我们点击多少次继续运行，他都一直在这里面，每次都会多出一个VMxx的标签，观察‘Call Stack’发现好像陷入了一个函数的递归调用中。这个‘debugger’让我们无法调试JS。但是只要关闭了F12窗口，网页就可以正常加载了

　　解决这种JS反调试的方法我们称之为’反-反调试’，其策略就是通过‘Call Stack’找到把我们带入死循环的函数，并且重新定义它

　　这样的函数几乎没有任何其他的功能，只是给我们设置的一个陷阱罢了。我们可以把这个函数在console中重新定义一下，比如将其定义成空函数，这样在运行的时候它就会什么都不干了，也就不会把我们带到无限递归的场景中去了，在这个函数调用的地方打个Breakpoint。因为我们已经在陷阱中了， 所以要刷新页面，JS的运行应该停止在设置的断点处，此时该函数还没有运行，我们就已经在console中重新定义完了，所以继续运行就会跳过该陷阱

第六天-爬取图片

一个网页的图片基本都是镶嵌在标签中，将标签里的链接提取并非易事，因此需要正则表达式。爬虫爬取图片的难度困扰就限制于正则表达式
难点：正则表达式

爬取糗事百科的图片

地址https://www.qiushibaike.com/imgrank/
通过浏览器调试台分析可知，糗事百科的图片加载均来自一个网络地址，其地址保存在html上，镶嵌在一个标签内。通过正则表达式提取其地址，之后在进行get请求可得到图片的文件，保存便得到了图片文件。
但是过程需要创建文件夹，以及对图片的命名，每次命名后保存图片所用的代码也需要改，这也是一个难点，或者说是值得关注的点。

    imgPath='./img/'+img_name

代码实战：

import requests

import re#正则
import os #创建文件夹用
#糗事百科图片分页爬取
import json
headers={
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) '
         'AppleWebKit/537.36 (KHTML, like Gecko) Chrome/85.0.4183.83 Safari/537.36 Edg/85.0.564.41'
}
#url="https://www.qiushibaike.com/imgrank/"
for page in range(2,13):
    pages=str(page)

    url = "https://www.qiushibaike.com/imgrank/page/" + pages + "/"

    response = requests.get(url=url, headers=headers)

    page_origin = response.text

    # 爬取图片链接
    # 所有图片所在标签ID为class

    # <div class="thumb">
    #
    # <a href="/article/123532830" target="_blank">
    # <img src="//pic.qiushibaike.com/system/pictures/12353/123532830/medium/76ZDSC6TAZI32FFB.jpg" alt="糗事#123532830" class="illustration" width="100%" height="auto">
    # </a>
    # </div>

    # 正则表达式编写

    ex = '<div class="thumb">.*?<img src="(.*?)" alt.*?</div>'
    img_list = re.findall(ex, page_origin, re.S)  # re.S 单行匹配 re.M多行匹配
    print(img_list)
    # 创建文件夹
    if not os.path.exists('./img'):
        os.mkdir('./img')

    for src in img_list:
        src = 'https:' + src  # 需要加协议头
        img_response = requests.get(url=src, headers=headers).content  # content表示二进制存储
        # 保存
        # 生成图片的名称
        img_name = src.split('/')[-1]

        imgPath = './img/' + img_name
        with open(imgPath, 'wb') as fp:
            fp.write(img_response)
            print("成功")

第20天-周记4

项目结束了！学习到很多，也留下了许多遗憾，最令我难忘的，其实并不是爬虫技术，python教程之类的，而是许多冷门的知识点，一个人如果没有学习过爬虫，根本不会了解何为user-agent，例如我们在用手机访问百度和用电脑访问百度得到的是不同的界面，原因就是因为user-agent告诉了网站我们所使用的浏览器和系统。若没有学习爬虫的相关知识，是很难知道这些东西的；再者如cookie，在之前我听到cookie这个单词，只会觉得他是“饼干”的意思，现在了解到cookie是一个登录网站很重要的东西，在我们输入完账号密码之后，网站会给浏览器一个cookie，相当于通行证，而浏览器会保存这个cookie，在下一次我们再次进入这个网站时，便无需登录就可以进入网站。当然学到的知识点是非常多的，包括但不限于浏览器F12调试、Ajax、post请求和get请求、post请求的表单（params）、selenium模拟等。这些都是非常好玩的东西，在学完爬虫之后，现在我每遇到一个网站，我可以清晰地看到这个网站所使用地技术，它地反爬能力高不高等特点。最后的项目虽然来说都是之前学的基础，但是真正编写起来还是会在很多地方无能为力，尤其在正则表达式的编写上，但是最后好在还是解决了这些问题。
